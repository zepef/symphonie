# Symphonie Project Memory

## What This Project Is
Symphony is a Next.js 16 App Router application that orchestrates Claude Code (or any JSON-RPC 2.0 agent) against Linear issues. It polls Linear, spawns per-issue agent subprocesses, manages concurrency/retries, and displays a live dashboard.

## Current State (Mar 9 2026)
All 8 improvement areas implemented. App is running. Full test suite passing (53 tests).

## Key Files
- `WORKFLOW.md` — config + LiquidJS prompt template (front matter = YAML config)
- `proxy.ts` — Next.js 16 auth middleware (renamed from middleware.ts; export fn is `proxy`)
- `instrumentation.ts` — boots orchestrator singleton on server start
- `tools/claude-shim.ts` — JSON-RPC shim bridging Symphony → `claude` CLI

## Architecture
- `lib/symphony/config/` — WORKFLOW.md parsing, env var resolution, hot-reload (chokidar)
- `lib/symphony/orchestrator/` — poll loop, state, dispatch, reconcile, retry, worker, index
- `lib/symphony/agent/` — subprocess.ts, rpc-transport.ts, event-parser.ts, session.ts, client.ts (re-export shim)
- `lib/symphony/tracker/` — Linear GraphQL adapter
- `lib/symphony/persistence/` — JSONL history store (`.symphony-history.jsonl`)
- `lib/symphony/notifications/` — fire-and-forget webhook
- `app/api/v1/` — state, health, [identifier] routes
- `app/api/auth/` — cookie login endpoint (sets `symphony_session` HTTP-only cookie)
- `app/login/` — login page
- `app/lib/api-fetch.ts` — browser fetch wrapper (credentials: include, redirects 401→/login)

## Auth
- `SYMPHONY_SECRET` env var enables auth; unset = open mode
- Accepts: Bearer header, `?token=` query, `symphony_session` HTTP-only cookie
- `/api/v1/health` excluded from auth (negative lookahead in matcher)

## Claude Code Shim (tools/claude-shim.ts)
Bridges Symphony JSON-RPC protocol to `claude --print --output-format stream-json`.
- Multi-turn: uses `--resume <sessionId>` from previous claude invocation
- Config: `CLAUDE_MODEL` (default: claude-sonnet-4-6), `CLAUDE_ALLOWED_TOOLS` (comma-sep)
- WORKFLOW.md: `codex.command: npx tsx tools/claude-shim.ts`

## Next.js 16 Conventions
- Middleware file is `proxy.ts` (not `middleware.ts`), export function named `proxy`
- Auth uses HTTP-only cookie session (NOT NEXT_PUBLIC_SYMPHONY_SECRET)
